Understanding Job Transitions

The tool that I built is called JobFit. It is a web application that identifies the skills deficit separating a user’s current job from their dream job.

The primary source of data I used to build this application was the O*NET database. The O*NET database is considered the nation’s primary source of occupational information. The database is quite extensive but static and difficult to use. However, it does serve as a useful starting place for investigating trends in jobs. The primary tables I used from this database were the list of Alternate Titles for jobs. This dataset lists the primary job title and a number of related job titles that fall into the same job category. I also used the Skills database which provides a list of 35 skill categories and for each category assigns a numerical score representing the importance of that skill for the each job title. Similarly, I use the Work Activities table which provides a list of 41 work activity categories and identifies a numerical score indicating the importance of each work activity to each job title. For some of the text-processing applications discussed below I also use the IWA and DWA tables which contain short descriptions of the work activities associated with the work activity categories.

In addition to the O*NET database I use data scraped from Indeed.com. I first wrote tool using BeautifulSoup and Selenium to scrape a selection of job posts given a search term and location (location options aren’t currently available). I collected meta-data from each post such as the job title, the location, the company name, the date it was posted and the text from the body of the post.

Lastly, I scraped resumes from Indeed.com. As I was constrained by time and by the amount of data Indeed would allow to be scraped before blocking my IP I only scraped the list of previous job titles from each resume. Effectively I collected a list of job transitions. This process has been slow and is still ongoing to improve the performance of my model.

The largest challenge in building my application was finding a way to identify or label job transitions as successful. My original approach was to treat the problem as unsupervised. I applied a k-means clustering algorithm to group the job titles based off of their skills profiles. I also experimented with other clustering algorithms, in particular, hierarchical clustering. I evaluated the performance of the two algorithms using the silhouette score. The silhouette score measures how similar an item is to its own cluster as compared to other clusters. A silhouette score can vary from -1 to +1 and a value close to 1 indicates that the item has been appropriately clustered. By averaging over all the data points in a cluster and average silhouette score can be computed and in practice a value above 0.5 indicates good clustering.

For my data, the average silhouette score for each cluster was ~0.2 and fairly insensitive to the number of clusters (k) and the clustering algorithm used. Therefore, I investigated other methods of evaluating clustering algorithms. I arrived at using “cluster stability” as a proxy for determining if I had chosen a correct value of k. I evaluate the cluster stability by randomly segmenting my data into subsets. For each subset I apply k-means clustering with a fixed value of k. I then find the number of clusters that minimizes the mean distance between clusters. Following this procedure I find that k=2 and k=5 provide stable clusterings.

I had hoped to use the cluster labels to determine which job transitions are possible; only transitions within clusters. Alternatively, I could measure the distance between two jobs in skill-space and categorize transitions based off of the distance (small distances being associated with possible transitions while large distances would be associated with difficult transitions). Both of these methods will be discussed below.

Some important questions I was asked at this point in my project were:

  1. How are you going to validate the job transitions you suggest?
  2. Aren’t some skills harder to learn than others?

These were roadblocks in my project until I was inspired to try to construct a labelled dataset of possible job transitions by using information contained in resumes. Resumes provide a chronology of a career trajectory. In particular, through the list of “previous job experience” they provide a list of job transitions that have been made in the past. By extracting this list of previous experience I began constructing pairs of jobs that individuals had transitioned between. If a job transition appeared on a resume I would label it 1 (success) and all other job transitions were labelled 0. I utilized a list of Alternate Titles from the O*NET database to perform keyword matching to match job titles from resumes to job titles I had skills profiles for contained in the O*NET database.

The idea to scrape data from resumes came relatively late into my project and as such have only obtained ~10,000 pairs of job transitions. This is only 1% of the total number of possible job transitions (~1,000,000 transitions obtained by taking all the possible permutations of pairs of job titles). As such I am dealing with a classic example of class imbalance; my success class is 100 times smaller than the other class.

The method by which I’m labelling successful transitions is somewhat incomplete. Given infinite time and a large database of resumes I could perhaps build the perfect training set where an unsuccessful transition necessarily corresponds to an unobserved transition. However, without more time to run my labelling algorithm I can’t distinguish between a transition that is unlikely to occur and a transition that I just haven’t come across yet in my data scraping.

The technique I employed to combat these problems was to undersample the majority class (unsuccessful transitions). I followed the following steps:

  1. Take bootstrap samples from the original population.
  2. Balance each sample by downsampling.
  3. Learn a model (i.e. logistic regression) from each sample.
  4. Majority vote.

The technique of downsampling helps to randomize the population that I label as an unsuccessful transition.
